# 01｜为什么我要用 AI 开发 Nginx Web 模块

最近我接近完成了一个难度比较高的 NGINX 任务。

这个过程中，我大量使用了 AI 辅助开发。结合自己的编程和工程经验，我保证了开发效率和代码质量。我认为自己应该算是会用 AI 开发了。

为了再次验证这套方法，我做了这个项目：

> https://github.com/hongzhidao/nginx-lua-web

这是一个从零开发的 Nginx Web 模块。我想做一个类似 Hono 那样的东西，让 Nginx 可以用 Lua 写 Web 应用。

从 Nginx 的角度看，它是一个比较原创的尝试。我对 Nginx、Lua、JavaScript 都有足够的经验，所以这个过程更多是让 AI 把我的想法兑现出来。

## 我对 AI 编程的理解

我对编程的理解是，它本质上是在对抗复杂性。开发是在构建一个系统，一步步逼近目标。

基于这个，我有几点认识。

第一，理解是基础，不管是对目标，还是对过程。它是对抗复杂性最好的方式之一。

第二，代码即设计。

所以我放弃了文档，比如 `AGENT.md`、`SPEC.md` 等。因为在开发过程中，我对目标和过程的理解是不断加深的，而代码是最能体现这些理解的东西。所以我推崇代码驱动，而 AI 编程就是 Pair Coding。

## 为什么选择 Nginx Web 模块？

因为这个项目难度很高，至少在 Nginx 里算中上偏难的。

难点在于我实现了完整的流式处理。请求和响应都是流式处理的，`fetch` 的请求和响应也要处理流式 body。

但我不知道它的实用价值，这得让市场去验证。我先把它做好，所以目前的价值更偏向知识：既有 AI 怎么开发一个完整项目，也顺便分享过程中碰到的编程经验，尤其是架构方面的经验。

我将把每个比较重要的阶段，按每篇文章去分享。
